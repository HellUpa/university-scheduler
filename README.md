# UCTTP Scheduler AI

**Интеллектуальная система автоматизированного составления университетских расписаний**

Программный комплекс для решения задачи планирования учебных занятий (University Course Timetabling Problem) с использованием гибридных генетических алгоритмов. Разработано в рамках дипломного проекта.

## 🚀 Основные возможности

- **Сравнение алгоритмов:** Реализованы два подхода — высокопроизводительный **Генетический алгоритм (GA)** и быстрый **Жадный алгоритм (Greedy)** для сравнительного анализа.
- **Матричный интерфейс:** Визуализация расписания в виде классической университетской матрицы (Группы / Временные слоты).
- **Движок правил (Rule Engine):** Модульная система оценки (Fitness Function), позволяющая легко добавлять новые ограничения (коллизии, окна, баланс нагрузки).
- **Адаптивная эволюция:** Применение импульсной мутации (Shock Therapy) и турнирной селекции для выхода из локальных оптимумов.
- **Реалистичный учебный план:** Поддержка системы кредитов, потоковых лекций, лабораторных работ и реальной сетки звонков.

## 🛠 Технологический стек

- **Backend:** Go 1.22+ (Fiber v2)
- **Database:** PostgreSQL 17 + GORM (ORM)
- **Frontend:** Vanilla JS + Tailwind CSS (Server-Side Rendering через Go Templates)
- **DevOps:** Docker, Docker Compose
- **Config:** Viper (управление окружением)

## 📦 Быстрый запуск

Для запуска всей инфраструктуры (Бэкенд + БД + UI) требуется только **Docker**.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/HellUpa/university-scheduler
   cd ucttp-backend
   ```

3. **Запустите систему:**
   ```bash
   docker-compose up --build
   ```

## 🖥 Использование

После запуска система доступна по адресу: **[http://localhost:8080](http://localhost:8080)**

### Веб-интерфейс позволяет:
- Настраивать гиперпараметры (размер популяции, количество поколений, шанс мутации).
- Запускать генерацию и видеть процесс "эволюции" в реальном времени.
- Сравнивать эффективность Генетического и Жадного подходов по Фитнес-метрикам и времени выполнения.
- Просматривать итоговое расписание в удобном матричном виде с подсветкой коллизий.

## 🧬 Особенности алгоритма

Система использует **Меметический Генетический Алгоритм**:
1. **Инициализация:** Эвристический подбор начальных позиций для минимизации Hard-конфликтов.
2. **Селекция:** Турнирный метод (Tournament Selection) для поддержания генетического разнообразия.
3. **Мутация:** Гибридная модель — плавная адаптация шанса мутации при стагнации и импульсный "шок" при глубоких локальных оптимумах.
4. **Оценка:** Мультипликативная модель фитнеса ($Fitness = HardScore \times (1 + SoftBonus)$), где ни один бонус за комфорт не может перевесить нарушение жесткого ограничения.

## 📊 API Endpoints

Для интеграции с внешними системами доступны следующие эндпоинты:
- `POST /api/v1/schedule/generate/genetic` — запуск генетической оптимизации.
- `WebSocket /api/v1/schedule/generate/genetic/ws` — запуск генетической оптимизации, с выводом процесса обучения
- `POST /api/v1/schedule/generate/greedy` — запуск жадной генерации.
- `GET /api/v1/health` — проверка состояния сервиса и БД.
