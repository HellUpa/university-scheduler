# Система Автоматизированного Составления Расписаний (UCTTP)

Бэкенд-сервис для решения задачи составления университетских расписаний (University Course Timetabling Problem) с использованием генетических алгоритмов.

Проект разработан в рамках дипломной работы.

## Стек технологий

- **Язык:** Go 1.22+
- **Веб-фреймворк:** Fiber
- **База данных:** PostgreSQL
- **ORM:** GORM
- **Конфигурация:** Viper
- **Контейнеризация:** Docker, Docker Compose

## Архитектура

Проект следует принципам чистой архитектуры с разделением на слои:
- **`cmd/api`**: Точка входа в приложение.
- **`internal/domain`**: Модели данных (GORM).
- **`internal/transport`**: HTTP-хендлеры (API).
- **`internal/algorithm`**: Реализация генетического алгоритма (основная бизнес-логика).
- **`internal/database`**: Управление подключением к БД, миграции и наполнение данными (seeder).

## Быстрый запуск

Для запуска проекта необходимо иметь установленные **Docker** и **Docker Compose**.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/HellUpa/university-scheduler.git
    cd university-scheduler
    ```

2.  **Запустите проект с помощью Docker Compose:**
    Эта команда соберет Go-приложение в легковесный Docker-образ, поднимет контейнер с PostgreSQL и запустит API.

    ```bash
    docker-compose up --build
    ```

    *Флаг `--build` необходим для пересборки образа при изменениях в коде.*

3.  **Проверка работоспособности:**
    - API будет доступен по адресу `http://localhost:8080`.
    - Проверить состояние сервиса можно, открыв в браузере `http://localhost:8080/api/v1/health`.

## Основной эндпоинт

- **`POST /api/v1/schedule/generate`**
  Запускает процесс генерации расписания. В теле запроса можно передать параметры для генетического алгоритма:
  
  *Без указания будут использованы значения по умолчанию*

  **Пример запроса (JSON):**
  ```json
  {
      "population_size": 200,
      "generations": 300,
      "mutation_rate": 0.05
  }
  ```
